fuckcapslock.js
Fuckcapslock is a JavaScript library which negotiates the effect of the CapsLock button — the case of the typed characters is not affected any longer, and the whole page behaves as if CapsLock was never active.
Why?

The CapsLock button has very attractive position — it is located in the home row (the one in the middle, where your hands lie relaxed) and is therefore easily accessible without bending the fingers. In fact, it's more accessible than any other non-letter key.

For historical reasons, the most accessible button has the most stupid function assigned to it — TO MAKE ALL THE TYPED CHARACTERS APPEAR IN CAPS. This rudimental side-effect made the button useless for other purposes.

Until now. With fuckcapslock.js the state of the CapsLock key is disregarded. There is yet an opprotunity to handle its events and reuse the button as a convenient hotkey or a modifier. It exposes the keyboard events in the normal way and can be identified by the key code 20 or code 'CapsLock'. Just like as this indicator reflects you pushing the CapsLock:

What exactly does fuckcapslock.js

It tracks the state of the CapsLock upon keyboard events, and if the CapsLock is on, it does the following:

  • Changing the content of the inputs, textareas and content-editable divs is prevented, the library then inserts the characters which would appear if the CapsLock was inactive;
  • Keyboard events (keyup, keydown and keypress) are taken over, suppressed, and re-emitted with updated properties as those would be if the CapsLock was inactive.
